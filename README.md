##Запуск и проверка webserver nginx
Сервер запускается автоматически при включении хостовой машины, при проблемах с сервером стоит ввести команду  "systemctl status nginx" и проверить ее вывод,
если nginx имеет статус "dead" требуется выполнить "systemctl start nginx" и выполнить команду curl https//localhost:8443/status в терминале должна
появиться информация о хосте.

##Cron выполнение скрипта
Скрипт запускается автоматически в 00:00 и собирает информацию о топ 3 ip по кол-ву запросов, общее количество запросов и успешные запросы (200).
Данный вносятся в базу данных log_analysis в таблицу log_stats.

##Решение проблемы при подключением с хоста и проверка 
Настроил сетевой мост и netplan, вошел по IP который узнал через ifconfig.
Проверка: введите в строку браузера https://<vm-ip_address>:8443/status

##Создание самоподписного сертификата
openssl req -x509 -nodes -days 365 -newkey rsa:4096 -keyout private.key -out certificate.crt
